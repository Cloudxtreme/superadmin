Laravel Events in Cloudwalkers project


http://laravel.com/docs/4.2/events

The main concept of Laravel Events and Triggers is followed in our project.
Anywhere in the code we can call/trigger some event by writting:
Event::fire(Config::get('cloudwalkers.trigger'), array($data));


There's one file that listen to all the events that are triggered across the system, that file is 
/app/events.php

This is where we listen and do something with the events, redirecting it to a controller@action where all the logic is placed
Event::listen('some.event', 'EventController@dealEvent');


This way we have the listenners centralized in one file which become easier to manage.
In order to test the events with input from console we can create Laravel commands like the one in
/app/commands/TriggerCommand.php

Commands are called from console with Artisan using the following syntax
php artisan command:trigger
