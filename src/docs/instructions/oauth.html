Registering an oauth app

OAuth1 registration tool: (DEPRECATED) https://devapi.cloudwalkers.be/oauth1/

OAuth2 registration tool: https://devapi.cloudwalkers.be/oauth2/register

OAuth2

In OAuth2, the redirect url of your application must always match the redirect url you send with the authorization request. So if you have multiple redirect urls (because, for example, you have a local development version and an online version), you must register a seperate oauth2 application for each environment.

More information on oauth2: http://oauth.net/2/

Auth flow

	website: hi, engine, can I please access you?
	engine, sure, but who are you?
	user: I'm Pedro, here is my password
	engine: alright, let me store that in the session for you. do you want me to share that info with website?
	user: yea, sure. That's the whole points...
	engine: well, alright. no need to be so annoyed about it
	user: sorry... I didn't mean it like that
	engine: you know, I have feelings too...
	user: ok. but can you get me back to the website now?
	engine. Sure. Hi, website, you can access pedro's details with this access token
	website: thanks

At that last point the engine doesn't care about the session anymore; website has it's access tokens.
