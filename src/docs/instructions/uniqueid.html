Unique IDs

In order to determine if two contacts or two messages are the same message, we use uniqueID fields in combination with grouping & groups.

Both contacts as messages have 3 fields: uniqueId, grouping & group. All 3 values must match for two entities to be considered equal.

Determine grouping & groupid

grouping can have 3 values: GLOBAL, SERVICE and STREAM.

GLOBAL

This option is used for all big networks.

In case of GLOBAL: ct_group contains the service_id (so 0-15). Example usage: all facebook contacts have a unique id, so for the service "facebook" all contacts have service_id = 3 and unique_id = xxx.

SERVICE

In case of service, ct_group contains the service id (so the id of the row in "services"). Example usage: drupal connections. For each drupal blog, user_id = 1 will contain a different contact. Multiple blogs can be connected to cloudwalkers, so a drupal connection with user 1 will have multiple values.

STREAM

In case of stream, ct_group contains the stream id the contact is in. This is mostly for backwards compatability from a time when messages and contacts were stored multiple times. At the moment I don't see any use of this one anymore./p>
