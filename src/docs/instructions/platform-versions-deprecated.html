"Tin Man"


rc2.cloudwalkers.be

Production roles:
Conversation Manager, Coworker

Contacts

A. Deliverables
  • Contact box
    • Contacts are clickable from inbox / accounts we follow / keyword monitoring
    • Information shown; Tweets / Posts (identify per network), Conversations, Contact notes
    • Add url of contact's profile on the social media channel
    • Show contact's avatar (in a correct size)
    • Able to add notes to the contact
    • Add note to content from the contact posts / tweets
    • When we add a note to content created by a contact, this note (+ copy of content) should also be attached to contact notes.
    • Post / Tweet actions available on tweets / posts from the contact on
    • View, edit and delete previous notes on a contact
  • List of contacts with notes
    • A list view of contacts with a note
    • Filter these contacts by name or network
B. Testing
  • Check if latest tweets / posts from contact are up to date
  • When no messages available from contact, clearly identify this towards the user
  • Add, edit, delete notes added to contact
  • If notes are correctly stored after re-login
  • Filters on the lists of contacts.

Notes

A. 3 types of Notes
  • Contact notes
    Information available in current contact box. (latest messages, conversations, notes, tags)
  • Content notes
    Related to content from messages / notifications, keyword monitoring, company accounts, accounts we follow
    When we add a note to content, this content should be displayed with the note.
    When we add a note to content, the contact box of the owner of this content needs to be updated.
    When content is webscraped, the content should be copied and added to the note.
  • General notes
    Notes attached to the account.
B. Deliverables
  • Notes Page
    • A list overview of all notes
    • Filter on; contact, creation date, word search
    • Possibility to see all contacts with notes
    • Notes should be related-grouped (like inbox messages)
    • Create a new note
  • Positioned
    • Add a new note from dropdown menu
    • Message in inbox; read, edit and add note to the content of the message.
    • Notification in inbox; read, edit and add note to the content of the message.
    • Coworkers message on wall; read, edit and add note to the content of the message.
    • Company accounts; read, edit and add note to the content of the message.
    • Accounts we follow; read, edit and add note to the content of the message.
    • Keyword monitoring; read, edit and add note to the content of the message.
  • Extra
    Add function private or share note to a new note, default is private.
    Share is between 1) Webcare group members, 2) Conversation Managers.
C. Testing
  • Check if notes in contact box is updated
  • Add note to contact in contact box
  • Add note to content from a message/notification; check if content is updated on note page and on contact detail
  • Add note to content from company accounts / accounts we follow; check if content is updated on note page and on contact detail
  • Create a new Note from navigation and Note Page
  • Edit notes, check if information is correctly saved and added to contact box.
  • Delete note
  • Filter creation date results on Note Page
  • Search on contact name on Note Page
  • Filter a list of all contacts with notes on Note Page
  • Check if privacy mode is correctly exeuted between profiles

"Copper Knot"


rc1.cloudwalkers.be

Production roles:
Conversation Manager, Coworker

Tags

A. 2 types of tags
  • Tags on content
    Tags added to notes, messages, notifications, posts/tweets company accounts (CA) & accounts we follow (AWF) and keyword monitoring (KM).

  • Tags on contact
    Tags added to a contact.
B. Deliverables
  • Able to add content tag to; Messages/notifications inbox, post/tweets company accounts & accounts we follow, keyword monitoring.
  • Able to add contact tag to contact in Contactbox
  • Able to add multiple tags to an item
  • Delete, add new tags of an item
  • Filter and sort tags on 'Notes Page'
  • Overview of contacts that are related to a tag.
  • Ability to set tags editable for certain user roles
C. Testing
  • Tags added to content and contacts
  • If tags are correctly stored after re-login
  • Edit, Delete tags from content and contacts
  • Check if filters on tagging are working correctly from ‘Notes Page’
