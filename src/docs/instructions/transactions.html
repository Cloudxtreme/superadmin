On MySQL transactions
Last updated: 2 months ago by Thijs V.

Because we're using gearman and long running php procedures, it is important to always handle mysql transactions correctly.

Because of this, always use try...catch when working with transactions.

 

$db = Database::getInstance ();

$db->start ();
try 
{
   // Do your business here
   $query = new Query ("UPDATE something FROM something WHERE something = ?");
   $query->bindValue (1, 'something');
   $query->execute ();

   // And commit
   $db->commit ();
}
catch (Exception $e)
{
   // First, rollback
   $db->rollback ();

   // Now, pass on the error
   throw $e;
}
